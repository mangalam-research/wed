A "pack" is a combination of:

  • Editing mode. Wed is a generic XML editor. Modes are largely what determines what features are available to the user. For instance, wed is bundled with a generic mode that can load just about any XML file but has no domain-specific knowledge. It would be conceivable to have a mode that offers specialized editing facilities. For instance, a mode could allow inserting links to a Zotero database of bibliographical elements.
  • Schema. We're talking about the XML schema that determines whether or not the document is valid.
  • Meta. This is an programmatic adjunct to the mode, which helps the mode customize its behavior. Schemas do not contain all the information that end-users care about. For instance, in a TEI schema, a "p" element should typically be rendered as a block, whereas a "hi" element should be represented inline. This information is not part of the schema and must be represented elsewhere. The meta takes care of this.
  • Metadata. The metadata performs a task similar to the meta but, whereas the meta operates dynamically, metadata is static. An example of metadata is documentation extracted from a schema.
